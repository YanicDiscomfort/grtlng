A small, low-level language I am working on. 'grtlng', of course, stands for "great language", and NOT (!!!!!!!) for "groot's language". Disney's Copyright is totally important to me and I would never try to use one of their characters in my own work.

I am planning for this to be able to compile to x86-64 (Linux only) and a 16-bit processor architecture I have yet to name (maybe "grtcmp"?) which I designed as a hobby project a while back.

This project was heavily inspired by the book "Crafting Interpreters" ([link](https://craftinginterpreters.com)) by Robert Nystrom, which has a [GitHub Repo](https://github.com/munificent/craftinginterpreters). It's a great book, regarding both information (seriously it's so understandable and exhaustive on the topic, the only thing not covered which I do here is code generation for a physical chip) and fun while reading.
